* For code files in this repo, do not read parts of them.  Always read the whole file into context.
* Add tests in `_test` packages
  - Prefer writing the test first, running it, confirming it fails as expected, then writing the code to make it pass.
  - The non-live test suite is very fast.  Run the full test suite by default instead of specific tests.
* commit message should include the standard `Co-Authored-By: <agent name> <agent email>` trailer, using the name and email
* When working with the OpenRouter API, read and familiarize yourself with https://openrouter.ai/docs/llms.txt and https://openrouter.ai/docs/api/reference/overview.md, and optionally other docs linked on those pages
* When working with the Artificial Analysis API, read https://artificialanalysis.ai/api/v2/openapi
